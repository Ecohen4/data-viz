# Shared-delhi-project/Delhi Data 2008 - 2013/Delhi .shp files/Consumption Choropleth/subdistricts 
# note: .shp, .shx, and .dbf all needed
delhi <- readShapePoly("map-data/subdistricts")

delhi <- fortify(delhi, id = "subdistrict") # convert to R dataframe for mapping
## Regions defined for each Polygons
# Change the IDs in the shape file to subdistrict name instead of numeric ID
# use subdivKey to do this 
for (i in unique(subdivKey$subdistrictID) ) {
  delhi$id[which(delhi$id == i)] <- subdivKey$subdistrict[which(subdivKey$subdistrictID == i)]
}
# identify the center of each subdivision (used to add labels to the map)
distcenters <- ddply(delhi, .(id), summarize, clat = mean(lat), clong = mean(long))

Questions and next steps

The final product will be entirely web-based, and will include an interactive D3 map visualization. The map will be value-by-alpha where census variables are the values, and energy use is the alpha. The user will be able to choose which census variable to visualize (one at a time). The user will also be able to toggle on/off the alpha filter for energy use, and mouse-over to reveal data values and subdistrict name.

I may also include a scatter plot, visualizing census variables against energy usage.

I am still deciding which census variables to use. I plan to review Sonia’s final thesis, to see what factors she identifies.

I am also interested to know if a measure of outages has been constructed from the raw data. If so, I’d like to include that measure as well.
